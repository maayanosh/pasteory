# PasteClone ‚Äî Plan & Research

Goal: a native macOS clipboard manager that closely replicates the look, feel, and core
functionality of **Paste** (pasteapp.io), running locally on the user's Mac (Apple Silicon,
macOS 26 / Darwin 25). No App Store, no sync, no accounts ‚Äî local only.

This document is self-contained. Execute milestones **in order**; each has verification
steps. Do not skip verification. Do not add features not listed here.

---

## 1. Research summary ‚Äî what Paste is and does

### 1.1 Signature UI (this is what "look and feel" means)

- **Bottom panel**: pressing the global hotkey (default **‚áß‚åòV**) slides a full-width panel
  up from the bottom of the screen, overlaying the current app *without* stealing focus
  from it visually (the previous app stays frontmost-looking). Pressing Esc or the hotkey
  again slides it back down.
- **Horizontal card timeline**: clipboard history rendered as large rounded cards in a
  horizontally scrolling row, newest on the left. Each card shows:
  - a **header strip color-coded by source app** (Safari items get one color, VS Code
    another, etc.) with the source app's name,
  - a large **content preview** (text excerpt, image thumbnail, link preview),
  - a footer with the **source app icon** and relative timestamp ("2 min ago").
- **Type-to-search**: just start typing while the panel is open ‚Üí search field activates,
  cards filter in real time. ‚åòF focuses search explicitly.
- **Pinboard tabs**: a tab strip above the cards: "Clipboard History" plus user-created
  pinboards. Pinboards hold items permanently; history can expire.
- **Quick paste numbers**: holding **‚åò** overlays numbers 1‚Äì9 on the first cards; ‚åò1..‚åò9
  pastes that item instantly.
- **Quick Look**: pressing **Space** on a selected card opens a large preview popover.
- **Frosted glass**: the panel background is a blurred translucent material (macOS
  "Liquid Glass" era styling ‚Äî use standard materials, they inherit the OS look).
- Keyboard-first: ‚Üê/‚Üí move selection, Return pastes selected, Esc closes.

### 1.2 Core behaviors

| Behavior | Detail |
|---|---|
| Capture | Everything copied anywhere: plain text, rich text, images, file URLs, links |
| Paste | Return/double-click pastes **into the app that was active before the panel opened** (simulated ‚åòV) |
| Plain-text paste | Hold ‚áß while pasting strips formatting |
| Copy without paste | ‚åòC on a selected card just puts it back on the clipboard |
| Pinboards | Create (‚áß‚åòN), drag/context-menu "Pin" items into them; items persist |
| Delete | Delete key removes selected item; context menu too |
| Privacy | Ignores concealed/password content; per-app exclusion list; pause capture toggle |
| Dedup | Re-copying an existing item moves it to the front instead of duplicating |
| Menu bar | Status item with Open, Pause, Settings, Quit |

### 1.3 Key technical facts (verified against Maccy, the open-source reference)

- macOS has **no clipboard change notification**. You must **poll
  `NSPasteboard.general.changeCount`** on a timer (~0.5 s / 500ms). This is what Maccy and every
  other clipboard manager does. Negligible CPU.
- Password managers mark secrets with pasteboard type
  **`org.nspasteboard.ConcealedType`** ‚Äî skip any pasteboard containing it. Also skip
  `org.nspasteboard.TransientType` and `org.nspasteboard.AutoGeneratedType`.
- Pasting into another app = write item to the pasteboard, then **synthesize ‚åòV with
  `CGEvent`**. This requires the **Accessibility permission** (System Settings ‚Üí Privacy
  & Security ‚Üí Accessibility). Prompt via `AXIsProcessTrustedWithOptions`.
- A **global hotkey** with no permissions needed: legacy **Carbon
  `RegisterEventHotKey`** ‚Äî still fully supported, used by most utilities. Do NOT use
  `NSEvent.addGlobalMonitor` for the hotkey (needs Accessibility and can't consume the event).
- The panel must be an **`NSPanel` with `.nonactivatingPanel`** style so the previous app
  keeps focus, BUT we need keyboard input in it ‚Äî so make it key (override
  `canBecomeKey = true`) while keeping the previous app's `NSRunningApplication` recorded
  so we can re-activate it before synthesizing ‚åòV.
- The app is a background/agent app: **`LSUIElement = true`** (no Dock icon), lives in the
  menu bar.
- For the TCC Accessibility grant to stick, the binary must live in a **stable, signed
  .app bundle** ‚Äî ad-hoc codesign is fine, but rebuilds must re-use the same bundle path
  and be re-signed (re-toggling the permission after rebuild is expected during dev).

## 2. Stack decision

**Swift + SwiftUI (UI) + AppKit (panel/window/pasteboard) via Swift Package Manager.**
No Xcode project file, no Electron, no third-party dependencies. Rationale: native look is
the whole point; SPM builds from the CLI (`swift build`); zero deps keeps a cheap model on
rails.

- Persistence: **SQLite via the built-in `SQLite3` C module** ‚Äî no; too fiddly.
  Use **plain JSON + files**: a `store.json` index plus per-item content files in
  Application Support. History cap 500 items. This is deliberately simple, atomic to
  implement, and plenty fast at this scale. (Do NOT introduce Core Data/SwiftData/GRDB.)
- Images stored as PNG files on disk; `store.json` holds metadata + text content inline.
- Target: macOS 14+ (`platforms: [.macOS(.v14)]`).

## 4. Data model

```swift
enum ClipKind: String, Codable { case text, richText, image, link, file }

struct ClipItem: Codable, Identifiable, Equatable {
    let id: UUID
    var kind: ClipKind
    var text: String?          // plain text content / URL string / file path
    var rtfFile: String?       // filename in content dir (rich text .rtf)
    var imageFile: String?     // filename in content dir (.png)
    var sourceAppBundleID: String?
    var sourceAppName: String?
    var createdAt: Date
    var pinboardID: UUID?      // nil = history
    var contentHash: String    // SHA256 of canonical content, for dedup
}

struct Pinboard: Codable, Identifiable { let id: UUID; var name: String; var colorHex: String }
```

Storage dir: `~/Library/Application Support/PasteClone/` ‚Üí `store.json`
(`{items: [ClipItem], pinboards: [Pinboard]}`) + `content/` for rtf/png files.
Save debounced (0.5 s) after mutations; atomic write. On item eviction/delete, remove its
content files.

Dedup rule: on capture, if an item with the same `contentHash` exists in **history**,
move it to front (update `createdAt`) instead of inserting. Pinned items are never
deduped against or evicted. History cap: 500 (evict oldest unpinned).

## 5. Component specs

### 5.1 ClipboardMonitor
- `Timer.scheduledTimer(withTimeInterval: 0.5, repeats: true)`; compare
  `NSPasteboard.general.changeCount` to last seen.
- On change: skip if paused, skip if pasteboard types contain any of the three
  `org.nspasteboard.*` markers above, skip if
  `NSWorkspace.shared.frontmostApplication?.bundleIdentifier` is in the exclusion list,
  and skip if the change was our own paste (set an `expectedChangeCount` in PasteService
  after writing).
- Read priority: file URLs (`readObjects(forClasses: [NSURL.self], options: [.urlReadingFileURLsOnly: true])`) ‚Üí
  `NSImage`/`.tiff`/`.png` data ‚Üí RTF (`.rtf` data, also grab plain string) ‚Üí plain
  `.string`. A string that is a single valid `http(s)` URL ‚Üí kind `.link`.
- Capture source app name + bundle ID from `frontmostApplication`.
- Ignore empty/whitespace-only strings and images > 20 MB.

### 5.2 HotKey (Carbon)
Wrap `RegisterEventHotKey` + `InstallEventHandler` for kEventHotKeyPressed. Default:
‚áß‚åòV (`keyCode 9` = V, modifiers `cmdKey | shiftKey`). Callback toggles the panel.
This is ~60 lines of well-known boilerplate; import Carbon.HIToolbox.

### 5.3 PanelController
- `NSPanel(contentRect:styleMask: [.nonactivatingPanel, .fullSizeContentView, .borderless], backing: .buffered, defer: false)`
- Subclass overriding `canBecomeKey = true`. `level = .statusBar`,
  `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`, `isOpaque = false`,
  `backgroundColor = .clear`, `hidesOnDeactivate = false`.
- Size: full screen width, height 360 pt, positioned at the bottom of the screen with
  the mouse (`NSScreen` containing `NSEvent.mouseLocation`).
- Content: `NSHostingView(rootView: PanelRootView(...))` whose background is
  `.ultraThinMaterial` with a top rounded-corner shape (radius 16) and a hairline top
  border (white 10% opacity).
- **Show**: record `NSWorkspace.shared.frontmostApplication` as `previousApp`; set frame
  offscreen (y = screenBottom ‚àí height), `orderFrontRegardless()`, `makeKey()`, animate
  frame up over 0.22 s with ease-out (`NSAnimationContext`, `allowsImplicitAnimation`).
- **Hide**: animate down 0.18 s, then `orderOut(nil)`.
- Esc hides (handle in `cancelOperation(_:)` or keyDown). Clicking outside hides
  (global mouse-down monitor while visible).

### 5.4 PasteService
```
paste(item, plainText: Bool):
  1. panel.hide()
  2. previousApp?.activate()
  3. write item to NSPasteboard.general (clearContents first; declare rtf+string,
     or png, or fileURL ‚Äî plainText mode writes only .string)
  4. record expectedChangeCount so the monitor ignores it
  5. after 120 ms delay (let the app regain focus):
       CGEvent keyDown/keyUp for keyCode 9 with .maskCommand, post to .cghidEventTap
```
On first use call `AXIsProcessTrustedWithOptions([kAXTrustedCheckOptionPrompt: true])`;
if not trusted, still copy to clipboard and show a menu-bar alert explaining the
permission (paste-in-place silently degrades to copy-only).

### 5.5 PanelRootView (SwiftUI)
Layout top-to-bottom:
1. **Header row** (height 44): PinboardTabs left ("History" + pinboards + "+" button),
   SearchBar right (250 pt wide, expands when focused).
2. **Card row**: `ScrollViewReader` + horizontal `ScrollView` + `LazyHStack(spacing: 12)`
   of `CardView`s, padding 16. Selection state = item id; ‚Üê/‚Üí moves it and scrolls to it;
   Return pastes; ‚áßReturn plain-pastes; Space toggles preview popover; Delete removes;
   ‚åò1‚Äì‚åò9 paste nth visible item; ‚åòF focuses search; ‚åòN new pinboard prompt.
   Typing any character while the list has focus routes into the search field.
3. Keyboard handling: use `.onKeyPress` (macOS 14+) on the focused container; monitor
   `.flagsChanged` via a local NSEvent monitor to toggle the ‚åò-number badges.

State object `AppState: ObservableObject` shared app-wide: `items`, `pinboards`,
`selectedTab`, `query`, `selection`, `isPaused`, filtered list computed (search matches
text, sourceAppName, kind; case-insensitive contains).

### 5.6 CardView ‚Äî replicate Paste's card precisely
- Fixed size **240 √ó 280 pt**, `RoundedRectangle(cornerRadius: 12)`,
  background `.regularMaterial`, subtle shadow (radius 8, y 2, black 25%).
- **Header** (height 36): filled with the item's app color; leading: source app name in
  11 pt semibold white (or black if the color is light); trailing: kind glyph
  (SF Symbol: `doc.text`, `photo`, `link`, `doc`).
- **Body**: text ‚Üí up to ~10 lines, 12 pt, top-leading, padding 12, monospaced if the
  text looks like code (contains `{`/`;`/4+ leading spaces). Image ‚Üí thumbnail
  `resizable().aspectRatio(contentMode: .fill)` clipped. Link ‚Üí URL host in bold +
  full URL below. File ‚Üí icon `NSWorkspace.shared.icon(forFile:)` + filename.
- **Footer** (height 30): app icon 16 pt (`NSRunningApplication` icon captured at copy
  time is NOT persistable ‚Äî fetch via `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)`
  ‚Üí `icon(forFile:)`, cache in memory), relative time ("now", "5m", "2h", "3d") in 10 pt
  secondary, pin indicator if pinned.
- **Selected state**: 3 pt accent-blue border + slight scale 1.02, animated 0.12 s.
- **‚åò held**: for the first 9 visible cards, a 22 pt rounded-square badge with the digit,
  top-right corner.
- Context menu: Paste, Paste as Plain Text, Copy, Pin to ‚ñ∏ (pinboards), Rename, Share ‚ñ∏,
  Delete.
- `AppColors.color(for: bundleID)`: fixed pleasant palette (10 hues); pick by
  `abs(bundleID.hashValue) % palette.count`, with hardcoded overrides for common apps
  (Safari‚Üíblue, Chrome‚Üíyellow, Finder‚Üísky, Notes‚Üíamber, VS Code‚Üíindigo, Slack‚Üípurple,
  Terminal/iTerm‚Üígray-black, Mail‚Üícyan, Xcode‚Üíblue).

### 5.7 PreviewPopover
On Space: a centered sheet-like overlay (max 70% panel width) showing full text
(scrollable, selectable), full-size image, or link with URL. Esc/Space closes.

### 5.8 SettingsView (opened from status-item menu, standard `NSWindow`)
- History limit (100/500/1000), Pause capture toggle, Excluded apps (add via
  `NSOpenPanel` on /Applications, list with remove), Launch at login
  (`SMAppService.mainApp.register()`), hotkey display (fixed ‚áß‚åòV ‚Äî changing it is out of
  scope), "Clear history" button.
- Persist settings in `UserDefaults`.

### 5.9 Status item
`NSStatusItem` with SF Symbol `doc.on.clipboard`. Menu: Open Paste (‚áß‚åòV), Pause/Resume
Capturing, Settings‚Ä¶, Clear History, Quit.

## 8. Gotchas the executor must respect

1. **Never poll faster than 0.3 s**; 0.5 s is the standard.
2. After rebuilding, macOS may treat the binary as changed ‚Üí **re-toggle Accessibility**
   for PasteClone in System Settings when paste-in-place stops working. Expected.
3. `.nonactivatingPanel` requires the panel class override `canBecomeKey`; without it
   no keyboard events arrive.
4. Set `expectedChangeCount` **before** the 120 ms paste delay, or the monitor recaptures
   the item and reorders history.
5. `NSHostingView` inside a borderless panel: set `panel.isMovableByWindowBackground = false`
   and give the SwiftUI root an explicit frame or the panel collapses to zero.
6. Don't store `NSImage` in the JSON ‚Äî files on disk, filenames in JSON.
7. All Store mutations on the main actor; the timer callback already runs on main.
8. When the panel is key and the user presses ‚áß‚åòV again, the Carbon hotkey still fires ‚Äî
   treat it as toggle (hide).
9. `activate()` on `NSRunningApplication`: use `activate(options: [])`; then the 120 ms
   sleep before CGEvent, else ‚åòV lands in the panel.
